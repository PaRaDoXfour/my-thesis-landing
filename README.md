# Інформаційна система інтелектуального керування бібліотечним каталогом
**Лендінг-сторінка бакалаврської роботи**

Цей репозиторій містить вихідний код офіційної веб-сторінки (лендінгу) для презентації бакалаврської роботи, розробленої студентом Сумського державного університету.

* **Виконавець:** Костян Олег
* **Група:** ІН-22.1
* **Тема:** Інформаційна система інтелектуального керування бібліотечним каталогом з підтримкою електронних та аналогових носіїв інформації

---

## 📖 Опис проєкту
У сучасному світі бібліотеки трансформуються у мультимедійні інформаційні центри. Дана система (презентована на цьому сайті) розроблена для управління гетерогенними колекціями, що включають:
* **Паперові видання** (звичайні та рідкісні)
* **Електронні книги** (різні формати)
* **Аудіовізуальні матеріали** 

## 🎯 Мета та завдання дослідження
Метою роботи є проєктування та розробка системи, що автоматизує облік та обслуговування користувачів з урахуванням специфіки різних типів носіїв.

**Основні завдання:** 
1. Аналіз предметної області та атрибутів медіа-ресурсів.
2. Проєктування ієрархії класів із використанням принципів ООП (Java).
3. Реалізація функціоналу додавання, пошуку та серіалізації даних.
4. Розробка інтерфейсу користувача.

## 🛠 Технологічний стек (Лендінг)
* **HTML5:** Семантична розмітка, ARIA-атрибути, SEO оптимізація (Open Graph, Schema.org).
* **CSS3:** Mobile-first адаптивний дизайн, CSS Grid, Flexbox, Custom Properties.
* **Зображення:** Оптимізація та використання `WebP` рішень.

## 🚀 Як запустити локально
Оскільки репозиторій є статичним веб-сайтом, для його перегляду достатньо:
1. Склонувати репозиторій:
   ```bash
   git clone https://github.com/PaRaDoXfour/my-thesis-landing.git
   ```
2. Відкрити файл `index.html` у будь-якому сучасному веб-браузері (подвійний клік по файлу).
3. *Або* використати локальний сервер для комфортної розробки (якщо встановлено Python):
   ```bash
   python3 -m http.server 8000
   ```
   Після чого перейти за адресою `http://localhost:8000`.

## 🔀 Git Стратегія
Проєкт розробляється з використанням стратегії **Release Flow (Microsoft Flow)**. Процес розробки включає:
* **Feature branches** для розробки окремих модулів або сторінок (наприклад, `feature/styling-responsive`).
* **Release branches** для фіксації стабільних версій.
* **Тегування** для позначення завершених етапів та релізів.
